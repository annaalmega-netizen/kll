# Автоматизация работы оператора грузоперевозок

Решение на PostgreSQL для автоматизации бронирования вагонов на грузовых
ж/д станциях. Клиенты (фирмы) заранее бронируют вагоны на станциях; станция
может обработать ограниченное число вагонов в сутки. Бронь невозможна при
превышении суточного лимита станции или при нехватке средств у клиента.
История операций сохраняется.

## Структура файлов

| Файл | Содержимое |
|------|-----------|
| `schema.sql` | Таблицы `stations`, `clients`, `bookings` (журнал операций) |
| `procedures_crud.sql` | Процедуры создания / изменения / удаления станций и клиентов (п.2) |
| `procedures_work.sql` | Рабочие операции `book_capacity`, `top_up_balance` (п.4) |
| `procedures_initdb_makejob.sql` | `initdb` (п.3) и `makejob` (п.5) |
| `functions_analytics.sql` | Аналитические функции (п.6) |
| `run_all.sql` | Главный скрипт: разворачивает всё и запускает демонстрацию |

## Запуск

```bash
createdb cargo
psql -d cargo -f run_all.sql
```

`run_all.sql` загружает все объекты, выполняет `initdb()` (очистка и
наполнение: 6 станций, 6 клиентов), `makejob()` (16+ рабочих операций,
включая срабатывание ограничений) и выводит три аналитических отчёта.

## Объекты БД

### Таблицы
- **stations** — наименование, макс. суточный объём (ваг.), стоимость
  отправки 1 вагона (руб.).
- **clients** — название, баланс денежных средств (руб.).
- **bookings** — журнал успешных бронирований (станция, клиент, дата,
  вагоны, стоимость).

### Процедуры (п.2 — по одной операции на объект)
`add_station`, `update_station`, `delete_station`,
`add_client`, `update_client`, `delete_client`.
Каждая выводит результат через `RAISE INFO`.

### Рабочие процедуры (п.4)
- **book_capacity(station_id, client_id, wagons, [date])** — проверяет
  суточный лимит станции и баланс клиента; при нарушении отменяет операцию
  и сообщает причину, иначе списывает средства и фиксирует бронь.
- **top_up_balance(client_id, amount)** — пополняет баланс клиента.

### Аналитические функции (п.6)
- **station_load_by_day(start, end)** — загруженность станций по дням
  (станция, дата, макс. объём, факт. объём, % загрузки).
- **station_work_volume(start, end)** — объём работы по станциям
  (станция, макс. объём, суммарный объём, средний объём в сутки).
- **client_rating(start, end)** — рейтинг клиентов по стоимости работы
  (название, кол-во станций, суммарный объём, стоимость, рейтинг).
